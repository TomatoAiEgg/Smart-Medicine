# 服务器环境

本文档记录当前用于开发联调的服务器、中间件容器和运维边界。历史上在云服务器运行过本项目容器，但当前开发不再使用云服务器承载本项目中间件或应用服务。

## 当前环境口径

- 应用服务：默认在 Windows 本地 IDEA / Maven 启动。
- 中间件：运行在老笔记本 Ubuntu，通过 Tailscale 地址访问。
- 本地 Docker Desktop、Redis、RocketMQ：默认不启动。
- 原云服务器：不再作为本项目运行环境，已清理本项目 `zhyf-*` 应用和中间件容器。

## 运维规则

- 本项目容器统一使用 `zhyf-` 前缀，避免误操作其他项目资源。
- 不复用其他项目 PostgreSQL、Redis、RocketMQ 容器和数据卷。
- 删除容器前先确认容器名、镜像、端口、数据卷归属。
- 数据卷默认保留，除非明确确认要清空数据。
- 本地开发优先直连 Tailscale 地址；如直连异常，再使用 SSH 隧道。
- 不因为本机无法连接就启动本地 Redis / RocketMQ / PostgreSQL。

## 应用服务运行策略

当前不在服务器上传 jar 或运行应用容器。后续 `order-service`、`message-service`、`gateway` 等应用服务默认在本地启动，并通过环境变量连接上述中间件。

历史上曾在云服务器临时运行过 `zhyf-order-service`、`zhyf-message-service` 用于第一阶段验证。该策略已废弃，相关容器已停止并删除。
