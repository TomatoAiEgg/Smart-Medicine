- 原云服务器：不再作为本项目运行环境，已清理本项目 `zhyf-*` 应用和中间件容器。

## 运维规则

- 本项目容器统一使用 `zhyf-` 前缀，避免误操作其他项目资源。
- 不复用其他项目 PostgreSQL、Redis、RocketMQ 容器和数据卷。
- 删除容器前先确认容器名、镜像、端口、数据卷归属。
- 数据卷默认保留，除非明确确认要清空数据。
- 本地开发优先直连 Tailscale 地址；如直连异常，再使用 SSH 隧道。
- 不因为本机无法连接就启动本地 Redis / RocketMQ / PostgreSQL。

## 4C8G 单机试点部署口径

后续如果只有一台 4 核 8GB 服务器，可以按“小单机试点/低并发生产”方案部署。该口径基于当前业务预估：

- 外部系统调用创建订单峰值约 `15 单/分钟`以内。
- 后台同时登录人数通常不超过 `5 人`。
- 煎煮、物流、回调、门户查单、报表和集成适配以低频人工操作或低频外部回调为主。

在这个负载下，4C8G 可以先承载前端、后端服务和必要中间件，但需要限制 JVM、数据库、中间件和日志资源，不能按高并发集群方案部署。

### 必须部署组件

| 组件 | 是否必须 | 说明 |
| --- | --- | --- |
| Nginx | 必须 | 承载 `admin-web` 静态文件，反向代理后端入口，后续配置 HTTPS |
| PostgreSQL | 必须 | 主业务数据库，所有核心表、Flyway migration 和联调数据都依赖它 |
| Redis | 建议必须 | 缓存、临时状态、后续锁和会话类能力；4C8G 下限制内存即可 |
| RocketMQ NameServer | 必须 | Outbox 事件发布和消费链路依赖 |
| RocketMQ Broker | 必须 | 订单事件、消息消费和异步补偿链路依赖 |
| 后端应用服务 | 必须 | 按业务服务分组启动，统一连接本机或内网中间件 |
| 日志轮转 | 必须 | 使用 Docker 日志限制或 `logrotate`，防止磁盘被日志打满 |
| 数据库备份 | 必须 | 至少每天备份 PostgreSQL；优先备份到独立磁盘或对象存储 |

### 可选组件

| 组件 | 建议 |
| --- | --- |
| Prometheus | 可后置；4C8G 上先不常驻，或只采集少量核心指标 |
| Grafana | 可后置；需要看板时再启 |
| Loki / Alloy / Promtail | 可后置；日志先依赖文件和 Docker 日志，避免占用内存和磁盘 |
| RocketMQ Dashboard | 排查时开启即可，不要求常驻 |
| cAdvisor / node-exporter | 可后置；如果开启，采集周期调低、保留时间调短 |

### 资源限制建议

| 组件 | 建议限制 |
| --- | --- |
| PostgreSQL | `shared_buffers` 控制在 `512MB` 左右，连接数先限制在 `50` 以内 |
| Redis | `maxmemory` 先设 `256MB`，淘汰策略按后续缓存用途再定 |
| RocketMQ Broker | JVM 堆先控制在 `512MB-1GB` |
| RocketMQ NameServer | JVM 堆先控制在 `256MB-512MB` |
| 单个 Java 服务 | JVM 堆优先控制在 `256MB-512MB`，核心服务可略高 |
| Nginx | 默认轻量配置即可 |
| 日志 | 单容器日志文件限制大小和数量，应用日志按天滚动并保留短周期 |

部署脚本或 Docker Compose 后续应显式设置 JVM 参数，例如：

```text
JAVA_TOOL_OPTIONS=-XX:MaxRAMPercentage=60 -XX:+UseG1GC
```

对内存紧张的服务可以改成固定堆：

```text
JAVA_TOOL_OPTIONS=-Xms128m -Xmx384m -XX:+UseG1GC
```

### 端口暴露原则

- 公网只暴露 `80/443`。
- 后端服务端口、PostgreSQL、Redis、RocketMQ、管理端口默认只允许本机或内网访问。
- 外部机构、医院、社康、物流需要回调时，统一走 Nginx / gateway 暴露的 HTTPS 地址。
- 如果外部系统要求 IP 白名单，使用服务器固定公网 IP。

### 保守容量目标

4C8G 单机按当前业务量可先按以下目标验收：

- 外部创建订单：`30 单/分钟`以内稳定。
- 后台在线用户：`5-10 人`。
- 日订单量：`1000-3000 单`稳定。
- 短期峰值：`5000 单/天`以内可接受，但需要观察数据库、RocketMQ 和 JVM 内存。

超过该范围后，优先拆分 PostgreSQL / RocketMQ 到独立机器，或把应用服务按核心链路与外围链路拆到不同服务器。

### 部署前准备清单

- 一台 Ubuntu 22.04 LTS 或 24.04 LTS 服务器。
- 固定公网 IP。
- 域名和 HTTPS 证书。
- SSH 登录账号，密码或私钥不得写入仓库。
- 确认磁盘空间，建议至少 `100GB`，更推荐 `200GB` 以上。
- 预留数据库备份位置，最好不要只放在同一块系统盘。
- 确认外部机构/医院/社康/物流是否需要公网 IP 白名单。
