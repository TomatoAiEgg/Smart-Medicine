# 新旧功能映射

本文档记录旧系统功能到新系统模块、页面、接口和服务能力的映射关系。

## 1. 映射原则

- 先保证旧系统能力可替代。
- 外部协议优先兼容，内部实现可以重构。
- 正常履约流程和人工修正流程分开建模。
- 机构、设备、物流、社康、地址等差异化逻辑通过适配器处理。
- AI 能力不进入第一版功能映射。

## 2. 应用级映射

| 旧应用 | 旧功能范围 | 新系统模块 |
| --- | --- | --- |
| `zhyfsupport` | 平台后台、审核、调剂、复核、物流、设备、统计、权限 | 平台管理后台 |
| `zhyf-api` | 机构 API、PDA API、物流 API、设备 API、定制 API | API 网关、机构适配器、设备适配器 |
| `zhyf-async` | MQ 消费、订单校验、审方、短信、回调 | 事件消费者、任务调度、回调服务 |
| `sc-zhyf-api` | 社康/智慧云/医院定制 API 变体 | 机构适配器、设备适配器 |
| `sc-zhyf-communityhospital` | 社康定时拉单、上传智慧药房、状态回写 | 社康集成服务 |
| `hospital-query` | 医院查单、物流查询、导出 | 机构门户 |
| `sc-address-input` | 地址补录、医院回推、微信 JSSDK | 地址补录门户、地址适配器 |

## 14. 暂缓功能映射

| 旧/新增能力 | 处理策略 |
| --- | --- |
| AI 提效 | 重构完成后独立规划 |
| 设备实时大屏 | 第一版不做大屏，只保留设备作业记录 |
| 复杂 BI | 第一版只做基础统计和导出 |
| 自动煎煮方案决策 | 第一版保留默认和规则建议，不自动决策 |
| 机构自助配置 | 第一版由平台后台配置 |
