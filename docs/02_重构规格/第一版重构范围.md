# 第一版重构范围

本文档用于明确第一版必须交付的最小生产闭环、暂缓功能、兼容边界和验收标准。正式落地前先以 `docs/03_架构设计/重构方案.md` 为总前提。

## 1. 第一版目标

第一版目标是替代旧系统的生产主链路。

第一版必须支持：

```text
机构推单
 -> 后台审核
 -> 调剂打单
 -> 复核
 -> PDA/设备煎煮
 -> 打包/面单/自提
 -> 物流回调
 -> 机构状态回调
 -> 医院查单
 -> 地址补录
 -> 社康拉单和状态回写
```

第一版不以新增 AI、BI、大屏、自动化调度为目标。

## 2. 必须交付范围

### 2.1 基础平台

| 能力 | 范围 |
| --- | --- |
| 机构管理 | 机构档案、接口密钥、回调地址、状态、所属煎煮中心 |
| 煎煮中心 | 中心档案、发货地址、仓储类型 |
| 用户权限 | 平台用户、机构用户、角色、菜单、接口权限、数据范围 |
| 字典配置 | 订单状态、处方类型、设备类型、特殊用法、物流配置 |
| 系统配置 | 短信模板、物流默认配置、接口开关、重试配置 |
| 操作日志 | 后台、PDA、接口、人工干预日志 |

### 2.2 订单与处方

| 能力 | 范围 |
| --- | --- |
| 机构推单 | 兼容旧 `/createOrder`，支持签名、白名单、幂等、重复处方校验 |
| 订单创建 | 写订单、处方、处方明细、接单轨迹 |
| 订单查询 | 平台后台查询、机构查询、按处方号查询 |
| 订单取消 | 兼容旧取消规则和状态码 |
| 地址修改 | 支持审核前地址修改，记录修改日志 |
| 处方修改 | 后台处方信息和明细修改，触发重新校验 |
| 状态日志 | 结构化记录每次订单状态变化 |

### 2.3 审核、调剂、复核

| 能力 | 范围 |
| --- | --- |
| 审核列表 | 查询待审核订单 |
| 审核详情 | 展示订单、处方、明细、地址、金额 |
| 审核保存 | 订单到 `15`，写审核人、时间、结果、轨迹、回调 |
| 订单锁 | 支持审核锁定和释放 |
| 调剂列表 | 查询待调剂处方 |
| 调剂打单 | 写调剂人、调剂时间、打印状态，返回打印数据 |
| 复核查询 | 校验处方可复核 |
| 复核保存 | 写复核人、时间、图片，代煎处方绑定水桶 |
| 复核完成 | 全部处方复核后订单到 `20` 并回调 |

### 2.4 PDA、设备、煎煮

| 能力 | 范围 |
| --- | --- |
| PDA 登录 | 兼容旧 PDA 登录 |
| 查询设备 | 查询可用煎煮机 |
| 绑定设备 | 处方绑定煎煮机，设备占用 |
| 开始煎煮 | 订单到 `25`，处方 `boil_status=1` |
| 完成煎煮 | 订单到 `27`，处方 `boil_status=2`，释放设备 |
| 取消/终止 | 支持 `operStatus=8/9` |
| 水桶绑定 | 复核时绑定加水桶 |
| 设备档案 | 水桶、煎煮机、PDA、打码机 |
| PDA 打码机关系 | 维护 PDA 与打码机关系 |
| 标签打印记录 | 保存和查询标签打印记录 |

### 2.5 物流与打包

| 能力 | 范围 |
| --- | --- |
| 发货列表 | 查询待发货订单 |
| 物流费用 | 兼容旧物流费用查询 |
| 创建物流单 | 顺丰/EMS 下单 |
| 面单查询/打印 | 查询面单数据，支持打印 |
| 打包完成 | 订单到 `30`，写运单、重量、包裹数、打包时间 |
| 自提完成 | 订单直接到 `40`，写出库/签收时间 |
| 顺丰回调 | 兼容 `36/37/40` 状态映射 |
| EMS 回调 | 兼容 `36/37/40` 状态映射 |
| 合并发货 | 多订单合并单、合并打包、合并回调 |
| 手工签收 | 单笔和批量签收 |

### 2.6 异步消息与回调

| 能力 | 范围 |
| --- | --- |
| 创建订单后处理 | 药品匹配、订单校验、处方校验、地址/位置校验、审方调用 |
| 状态回调 | 订单状态 `15/20/27/30/40/98/99` 等按旧规则回调 |
| 短信消息 | 审核通过、发货等短信发送 |
| 回调记录 | 保存请求、响应、状态、重试次数 |
| 消费记录 | 保存消息消费结果 |
| Outbox | 业务事务和事件发送解耦 |

### 2.7 社康集成

| 能力 | 范围 |
| --- | --- |
| 区域任务 | 支持旧深圳各区定时拉单 |
| 本地消息 | 保存社康消息防漏单 |
| 订单详情拉取 | 调社康接口获取订单 |
| 订单转换 | 转换成智慧药房订单 |
| 上传智慧药房 | 调新系统内部订单创建能力 |
| 状态回写 | 上传成功后回写社康 |
| 失败重试 | 回写失败记录和定时重试 |
| 退费回写 | 支持退费状态回写 |

### 2.8 医院查单

| 能力 | 范围 |
| --- | --- |
| 医院登录 | 医院用户登录和权限 |
| 数据范围 | 按机构授权范围查询 |
| 订单列表 | 以处方维度查询订单 |
| 订单详情 | 订单、处方、明细 |
| 物流轨迹 | 展示履约/物流轨迹 |
| 导出 | 导出订单列表 |

### 2.9 地址补录

| 能力 | 范围 |
| --- | --- |
| 统一入口 | 按机构跳转页面 |
| 医院页面 | 支持旧系统已有医院页面 |
| 地址查询 | 查询历史地址 |
| 地址保存 | 保存或更新患者地址 |
| 地址回推 | 按医院协议推送 |
| 推送日志 | 保存请求、响应、状态 |
| 防重复提交 | Redis 或等效机制 |
| 微信 JSSDK | 保留签名能力 |

### 2.10 人工修正

| 能力 | 范围 |
| --- | --- |
| 初始化处方 | 订单回 `5`，释放设备，绑定记录失效 |
| 取消处方 | 支持单处方取消和整单取消 |
| 订单走流程 | 人工补齐全流程并到 `40` |
| 人工签收 | 单笔和批量签收 |
| 审计记录 | 所有人工修正必须记录原因、操作人、时间 |

## 3. 第一版暂缓范围

| 功能 | 暂缓原因 |
| --- | --- |
| AI 提效能力 | 必须等核心业务和数据闭环稳定后再做 |
| 自动审方决策 | 医疗责任高，第一版只保留审方平台/规则辅助 |
| 自动煎煮方案决策 | 需要设备数据和规则验证 |
| 大屏驾驶舱 | 不影响旧系统替代 |
| 复杂 BI 报表 | 第一版只保留必要统计和导出 |
| 设备实时监控大屏 | 先保留设备档案和作业记录 |
| 多租户商业计费 | 第一版聚焦业务迁移 |
| 机构自助配置 | 第一版由平台后台配置 |
| 全量 UI 视觉重设计 | 不作为核心验收标准 |

## 4. 第一版兼容边界

### 4.1 必须兼容

- 旧机构推单协议。
- 旧订单状态码。
- 旧 PDA 接口能力。
- 旧物流回调状态映射。
- 旧医院查单的数据范围。
- 旧社康拉单和回写流程。
- 旧地址补录医院差异协议。

### 4.2 可以内部重构

- 数据表结构。
- 服务模块边界。
- 消息中间件实现。
- 状态日志实现。
- 回调重试实现。
- 前端后台页面结构。

### 4.3 需要配置化

- 机构密钥。
- IP 白名单。
- 回调地址。
- 物流产品。
- 社康区域接口地址。
- 医院地址补录协议。
- 短信模板。
- 订单状态回调规则。

## 5. 验收标准

### 5.1 主链路验收

| 链路 | 验收标准 |
| --- | --- |
| 机构推单 | 能创建订单、处方、明细，重复推单能拦截 |
| 审核 | 审核通过后订单到 `15`，机构收到回调 |
| 调剂 | 调剂打单后写调剂信息和打印状态 |
| 复核 | 复核完成后订单到 `20`，机构收到回调 |
| 煎煮 | PDA 开始/完成后订单到 `25/27`，设备占用释放正确 |
| 打包 | 打包后订单到 `30`，运单和重量正确 |
| 物流 | 回调后订单到 `36/37/40`，轨迹完整 |
| 查单 | 医院用户只能查授权机构订单 |
| 地址补录 | 地址可保存并回推医院，失败有日志 |
| 社康 | 社康订单可拉取、上传、回写，失败可重试 |

### 5.2 数据验收

- 每个订单状态变化有结构化日志。
- 每个关键人工操作有审计。
- 每次机构回调有记录。
- 每次物流回调有原始记录和归一状态。
- 每条社康消息有处理状态。
- 每次地址回推有请求和响应日志。

### 5.3 异常验收

- 重复处方拦截。
- 回调失败可重试。
- 社康状态回写失败可重试。
- PDA 重复上报不导致状态错乱。
- 物流状态不倒退。
- 设备不会被并发重复占用。
- 人工修正必须有审计。

## 6. 第一版交付文档要求

第一版开发前至少需要完成：

- `新旧功能映射.md`
- `新旧接口映射.md`
- `新旧数据表映射.md`
- 数据库设计文档。
- 状态机设计文档。
- 接口兼容说明。
- 测试验收用例。

## 7. 当前结论

第一版重构范围已经覆盖旧系统生产主链路和必要外围接入。

后续开发应按“主链路优先、外围兼容次之、增强能力最后”的顺序推进。
