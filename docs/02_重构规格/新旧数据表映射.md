# 新旧数据表映射

本文档记录旧数据库表到新 PostgreSQL 表的领域归属、字段映射方向、迁移规则和废弃字段。

当前为重构设计阶段映射，不是最终 DDL。

## 1. 映射原则

- 旧表按领域重新拆分，不直接一比一搬表。
- 订单、处方、履约、物流、设备、回调、日志分开建模。
- 旧系统字符串状态历史 `order_status_follow` 迁移为结构化状态日志。
- 外围集成表统一到集成任务、回调记录、适配器配置。
- 敏感字段如密码、密钥、Token 不进入普通业务表明文展示。

## 2. 核心订单表

### 2.1 `zhyf_order`

旧表职责：

- 订单主表。
- 同时承载收货地址、物流、包裹、状态、回调地址、机构等信息。

建议新表：

| 新表 | 职责 |
| --- | --- |
| `order_main` | 订单主信息 |
| `order_address` | 收货地址快照 |
| `order_status_log` | 结构化订单状态日志 |
| `order_callback_config_snapshot` | 创建订单时的回调配置快照，可选 |
| `shipment` | 物流单/包裹信息 |
| `shipment_trace` | 物流轨迹 |

核心字段映射：

| 旧字段 | 新归属 | 说明 |
| --- | --- | --- |
| `order_id` | `order_main.order_no` | 保留旧订单号 |
| `sp_orderid` | `order_main.external_order_no` | 旧系统默认等于订单号 |
| `company_num` | `order_main.institution_code` | 机构编号 |
| `storage_type` | `order_main.decoction_center_code` | 煎煮中心/仓储类型 |
| `order_status` | `order_main.status` | 对外状态码保留 |
| `order_status_follow` | `order_status_log` | 拆成多行状态日志 |
| `order_time` | `order_main.order_time` | 机构订单时间 |
| `callback_url` | `order_main.callback_url` | 可保留快照 |
| `province/city/zone/addr_detail` | `order_address` | 收货地址 |
| `consignee/tel` | `order_address` | 收货人和手机号 |
| `delivery_time` | `order_address.appointment_time` | 预约配送时间 |
| `addr_type` | `order_address.address_type` | 自提/配送 |
| `logistics_*` | `shipment` | 物流字段拆出 |
| `package_time/outbound_time/sign_time` | `shipment` | 打包/出库/签收时间 |
| `order_remark` | `order_main.remark` | 备注 |
| `version` | `order_main.version` | 乐观锁 |

迁移注意：

- 旧表中物流字段和订单字段耦合严重，新库应拆分。
- 旧状态流转字符串不可靠，迁移时可按字符串拆历史，也要保留原文。

## 13. 废弃或归档字段

第一版不直接删除旧字段含义，但新表设计中可弱化：

| 旧字段/模式 | 处理 |
| --- | --- |
| `order_status_follow` | 迁移到状态日志，原文归档 |
| 混合在订单表的物流字段 | 拆到物流表 |
| 混合在处方表的作业字段 | 拆到作业记录 |
| 调试日志输出字段 | 不迁移或归档 |
| 明文密钥输出 | 禁止迁移到可读日志 |

## 14. 迁移批次建议

### 14.1 基础配置

先迁移：

- 机构。
- 煎煮中心。
- 用户角色权限。
- 字典。
- 系统配置。
- 设备。
- 物流配置。

### 14.2 业务主数据

再迁移：

- 订单。
- 处方。
- 处方明细。
- 物流单。
- 合并发货。

### 14.3 历史和日志

最后迁移：

- 履约轨迹。
- 操作日志。
- MQ/回调记录。
- 地址补录记录。
- 社康消息。
- 设备作业记录。

## 15. 当前结论

新库不建议直接复制旧库结构。

第一版应按领域拆表，但必须保证旧订单号、处方号、机构编号、状态码、运单号、回调记录和日志可追溯。
