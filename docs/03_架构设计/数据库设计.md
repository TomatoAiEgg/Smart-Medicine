# PostgreSQL 数据库设计

## 1. 设计原则

- 按领域建模，不照搬旧表。
- 每张核心业务表都带 `tenant_id`。
- 高增长日志/轨迹/事件表按月分区。
- 状态变化必须有日志表。
- 外部请求、回调、消费、重试必须有独立记录。
- 业务主数据和历史日志分离。

## 12. 当前结论

第一版数据库设计的关键不是字段多，而是领域清楚、日志完整、状态可追溯、回调可补偿。
